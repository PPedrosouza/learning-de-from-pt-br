# Sons, alfabeto e nomes técnicos

Nível: A1

## Objetivo

Ao final deste tópico, você será capaz de reconhecer as letras do alfabeto em inglês, soletrar informações profissionais simples e pedir repetição quando não entender nomes, e-mails, nomes de usuário, arquivos ou siglas técnicas como API, UI, URL e SQL.

## Explicação

Em ambientes de trabalho com software, é comum precisar soletrar dados curtos: nome, e-mail, usuário do GitHub, nome de arquivo, branch ou sigla. Em inglês, a pergunta mais útil para isso é:

**Can you spell that?**

Use essa frase quando você ouviu uma palavra, mas precisa das letras. Para responder, diga as letras separadamente:

**S-E-M**

Para introduzir a soletração, você pode usar:

- **It is spelled...** = soletra-se...
- **My email is...** = meu e-mail é...
- **My GitHub username is...** = meu usuário do GitHub é...
- **The file name is...** = o nome do arquivo é...

Algumas letras confundem brasileiros porque o som não parece com o nome em português:

| Letra | Som aproximado para brasileiros | Observação |
| --- | --- | --- |
| A | ei | Como em **API**: A-P-I. |
| E | ii | Não é o som de “é” em português. |
| I | ai | Como em **UI**: U-I. |
| Y | uai | Muito comum em nomes e arquivos. |
| W | dâbliu | Duas letras em português, uma letra em inglês. |
| H | eitch | Não é “agá”. |
| R | ar | Em siglas, diga o nome da letra. |
| U | iu | Como em **URL**: U-R-L. |

Em siglas técnicas, normalmente dizemos letra por letra:

- **API**: A-P-I
- **UI**: U-I
- **URL**: U-R-L
- **SQL**: S-Q-L, embora algumas pessoas digam “sequel”

Para nomes de arquivo, diga o ponto como **dot** e o hífen como **dash**:

- **app.js** = app dot J-S
- **user-data.csv** = user dash data dot C-S-V

## Exemplos

| Inglês | Tradução | Observação |
| --- | --- | --- |
| Can you spell that? | Você pode soletrar isso? | Pedido educado e direto. |
| My email is ana dot silva at example dot com. | Meu e-mail é ana.silva@example.com. | Use **dot** para ponto e **at** para arroba. |
| It is spelled M-A-R-I-A. | Soletra-se M-A-R-I-A. | Use letras separadas por hífen na escrita. |
| My GitHub username is dev dash sem. | Meu usuário do GitHub é dev-sem. | **dash** indica hífen. |
| The file name is index dot H-T-M-L. | O nome do arquivo é index.html. | Letras de extensão costumam ser soletradas. |
| Is that U-I or U-R-L? | Isso é UI ou URL? | Útil para confirmar siglas parecidas. |

## Erros comuns

- Dizer **I** com som de “i” em português. Em inglês, a letra **I** soa como **ai**.
- Confundir **E** e **I**. **E** soa como **ii**; **I** soa como **ai**.
- Ler **W** como “dáblio” em contexto internacional. Prefira aproximar de **double u**.
- Esquecer **dot** e **at** ao falar e-mails.
- Tentar traduzir siglas técnicas. Em geral, diga as letras em inglês.

## Prática guiada

1. Soletre seu primeiro nome em inglês.
2. Soletre um e-mail profissional usando **dot** e **at**.
3. Pergunte a alguém: **Can you spell that?**
4. Diga as siglas **API**, **UI**, **URL** e **SQL** letra por letra.
5. Soletre um arquivo simples, por exemplo: **main.py** ou **index.html**.

## Resumo

Para sobreviver a primeiras conversas profissionais em inglês, domine as letras mais confusas, use **Can you spell that?** quando precisar de ajuda e pratique nomes técnicos frequentes. Não é necessário falar rápido: clareza é mais importante do que velocidade.
