# História: Sons, alfabeto e nomes técnicos

Nível: A1

## História em inglês

Ana is new on the support team.

Tom asks, "What is your email?"

Ana says, "It is ana dot silva at company dot com."

Tom asks, "Can you spell that?"

Ana says, "A-N-A dot S-I-L-V-A at company dot com."

Tom says, "Thank you. What is your GitHub username?"

Ana says, "It is ana dash dev."

Tom asks, "Is the file name app dot J-S?"

Ana says, "Yes. The API is ready, and the UI is simple."

Tom says, "Great. Can you repeat the URL?"

Ana says, "Yes: docs dot company dot com."

## Tradução linha a linha

| Inglês | Tradução |
| --- | --- |
| Ana is new on the support team. | Ana é nova na equipe de suporte. |
| Tom asks, "What is your email?" | Tom pergunta: "Qual é o seu e-mail?" |
| Ana says, "It is ana dot silva at company dot com." | Ana diz: "É ana.silva@company.com." |
| Tom asks, "Can you spell that?" | Tom pergunta: "Você pode soletrar isso?" |
| Ana says, "A-N-A dot S-I-L-V-A at company dot com." | Ana diz: "A-N-A ponto S-I-L-V-A arroba company ponto com." |
| Tom says, "Thank you. What is your GitHub username?" | Tom diz: "Obrigado. Qual é seu nome de usuário do GitHub?" |
| Ana says, "It is ana dash dev." | Ana diz: "É ana-dev." |
| Tom asks, "Is the file name app dot J-S?" | Tom pergunta: "O nome do arquivo é app.js?" |
| Ana says, "Yes. The API is ready, and the UI is simple." | Ana diz: "Sim. A API está pronta, e a UI é simples." |
| Tom says, "Great. Can you repeat the URL?" | Tom diz: "Ótimo. Você pode repetir a URL?" |
| Ana says, "Yes: docs dot company dot com." | Ana diz: "Sim: docs.company.com." |

## Vocabulário da história

| Inglês | Tradução | Observação |
| --- | --- | --- |
| support team | equipe de suporte | Contexto profissional comum. |
| email | e-mail | Pode ser dito com **at** e **dot**. |
| GitHub username | nome de usuário do GitHub | Útil em vagas e projetos. |
| file name | nome de arquivo | Frequentemente inclui **dot**. |
| API | API | Diga A-P-I. |
| UI | interface de usuário | Diga U-I. |
| URL | URL | Diga U-R-L. |

## Perguntas de compreensão

1. Qual é o e-mail de Ana?
2. O que Tom pede para Ana soletrar?
3. Qual é o nome de usuário do GitHub de Ana?
4. Qual nome de arquivo Tom confirma?
5. A API está pronta?
